# Handoff 1.4b — Productos (CRUD completo)

**Estado:** Completada ✅

## Objetivo alcanzado
Migración `productos` ejecutada. CRUD completo con búsqueda/filtro por nombre, código, categoría y estado, paginación, eliminado lógico (estado=eliminado), duplicar producto, subida de imagen. 7 rutas protegidas por middleware granular.

- `productos` (id, codigo, nombre, descripcion, categoria_id FK, marca, precio_compra, precio_venta, stock, stock_minimo, imagen, estado enum, timestamps)

## Decisiones tomadas
- Mismo patrón que categorías: middleware `permiso:productos.*` en cada ruta, sin `$this->authorize()` en controller
- Vista form con `isset($producto)` (NO pasa new Producto vacío en create, coincidiendo con patrón de categorías que sí pasa new Categoria)
- Stock no editable en form (solo informativo), conforme regla 7
- Duplicar: código único (ORIGINAL-COPIA, ORIGINAL-COPIA-2…), stock=0, estado=inactivo
- Imagen: storage/public con cleanup al actualizar
- Vista index usa Tailwind CSS, permisos con helper `auth()->user()->role?->tienePermiso()`, script inline para preview de imagen
- `scopeActivos()` en modelo para filtrar eliminados en listado
- `scopeStockCritico()` para badge rojo cuando stock ≤ stock_minimo

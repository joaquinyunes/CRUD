# Handoff 1.2b — Middleware de permisos + Vistas de gestión de roles

**Estado:** Completada ✅

## Objetivo alcanzado
Middleware `CheckPermission` (alias `permiso`) creado con bypass automático para Administrador y soporte OR de múltiples permisos. `RoleController` con índice, edición de permisos por módulo, y asignación de roles a usuarios. Tres vistas Blade. Archivo de rutas `routes/roles.php` con 5 rutas protegidas por `auth` + `verified` + `permiso`.

## Archivos creados
- `app/Http/Middleware/CheckPermission.php`
- `app/Http/Controllers/RoleController.php`
- `routes/roles.php`
- `resources/views/roles/index.blade.php`
- `resources/views/roles/edit.blade.php`
- `resources/views/roles/usuarios.blade.php`

## Archivos modificados
- `bootstrap/app.php` (alias `'permiso' => CheckPermission::class`)
- `routes/web.php` (`require __DIR__.'/roles.php'`)

## Decisiones tomadas
- Administrador bypasa el middleware (acceso total, sin consulta a BD)
- Middleware acepta múltiples permisos como OR
- Edición de permisos del Administrador bloqueada (controlador + vista)
- Usuario no puede quitarse su propio rol
- Permisos agrupados por módulo con toggle "Seleccionar todos"

## Microfase 1.2 completa
1.2a (migraciones + modelos + seeder) + 1.2b (middleware + vistas) = 1.2 terminada.

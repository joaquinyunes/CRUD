# Handoff 1.3 — Dashboard base + Layout

**Estado:** Completada ✅

## Objetivo alcanzado
Layout propio reemplazando el de Breeze: sidebar con navegación por módulos (con permisos), navbar con datos del usuario y botón de salir, flash messages. Dashboard con métricas de "Hoy" (ventas, ingresos, clientes — placeholders hasta fases 2.1/1.5) y "Sistema" (usuarios — dato real, productos y stock — placeholders hasta fase 1.4). Raíz `/` redirige a dashboard.

## Archivos creados
- `app/Http/Controllers/DashboardController.php`
- `resources/views/layouts/app.blade.php` (reemplaza el de Breeze)
- `resources/views/layouts/partials/sidebar.blade.php`
- `resources/views/dashboard.blade.php` (reemplaza el de Breeze)

## Archivos modificados
- `routes/web.php` (redirect `/` a dashboard, ruta `/dashboard` a `DashboardController`)

## Decisiones tomadas
- Layout usa `@extends('layouts.app')` y `@yield('content')`, no `<x-app-layout>` (separación de Breeze)
- Sidebar verifica permisos con `tienePermiso()` antes de mostrar cada enlace
- Módulos futuros (productos, ventas) con `href="#"` y `TODO` hasta su microfase
- Único dato real: `User::count()` — los demás son `0` con TODO documentado
- Layout de Breeze (`navigation.blade.php`) no se elimina pero ya no se usa
